# Version 4.7 (The Security & Auth Expansion)

## Overview
Version 4.7 locks down the entire Calc-Avg application by introducing a robust Security & Authentication Gateway natively powered by the C server. Previously open access to surveillance capabilities (Screen Streaming, Keylogging, Audio, API endpoints) has now been strictly secured.

## Key Features & Improvements
1. **Authentication Gateway**
   - Engineered a secure login portal located at `web-app/login.html` leveraging the premium Glassmorphism aesthetic.
   - Introduced a native `/api/login` route in the C server that validates credentials.
   
2. **Cryptographic Session Management (SHA-256)**
   - Deployed the native C implementation of SHA-256 (`crypto.c`) to generate irreversible cryptographic tokens upon successful authentication.
   - Session tokens are stored in the server's heap memory and tracked asynchronously utilizing Windows `CRITICAL_SECTION` synchronization.
   
3. **HTTP Cookie Routing & Security**
   - The C server now inherently parses incoming `Cookie:` headers.
   - Any unauthenticated HTTP request targeting private APIs or the dashboard is immediately redirected via `HTTP 302 Found` to the login gateway.
   
4. **WebSocket Connection Filtering**
   - Implemented strict handshake interception during the `Upgrade: websocket` protocol negotiation.
   - WebSockets attempting to establish the high-speed 60FPS surveillance stream without a valid session token are aggressively dropped with a `401 Unauthorized` response.
