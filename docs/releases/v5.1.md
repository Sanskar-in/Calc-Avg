# Release Notes - Version 5.1.0.0 (The POSIX Hardening Update)

## Overview
Version 5.1 represents the ultimate architectural hardening of the C Engine backend. While V4.9 successfully abstracted the WinSock API to UNIX Sockets, the core 8-worker Thread Pool Queue algorithm still relied deeply on Windows Kernel Synchronization Primitives (`CONDITION_VARIABLE`, `SleepConditionVariableCS`, `WakeConditionVariable`). This prevented true cross-platform Linux compilation.

## Key Features & Changes
- **POSIX Thread Pool Hardening:** Completely ripped out the unabstracted `CONDITION_VARIABLE` code paths from the API Queue loop. 
- **UNIX Deadlock Prevention:** Secured the worker threads with a robust `#ifdef _WIN32` abstraction layer that cleanly wraps the queue state with standard POSIX `pthread_cond_t` and `pthread_mutex_t` APIs.
- **WebSocket Thread Safety:** Abstracted the rogue `CreateThread` call used to spawn the real-time Remote Desktop broadcasting thread with `pthread_create`, eliminating the final undefined reference blocking compilation on Ubuntu/macOS.

*Calc-Avg is now fully POSIX-compliant and cross-platform native. Signed off by Sanskar - 2026*
