# Version 4.6 (The Binary Streaming Expansion)

## Overview
Version 4.6 introduces a complete overhaul of the WebSocket engine, upgrading the transmission from Base64 text-encoded strings to high-performance Binary Frames (Opcode `0x82`). This drastic architectural change eliminates significant overhead, empowering the server to deliver a buttery-smooth 60FPS surveillance and screen-streaming experience without compromising CPU resources. 

Additionally, we've integrated a real-time, two-way Walkie-Talkie audio functionality using Windows Multimedia (`winmm.lib`) and HTML5 Web Audio API, establishing a true full-duplex communication channel.

## Key Features & Improvements
1. **High-Performance Binary WebSocket Streaming**
   - Transformed the WebSocket engine to use binary data formatting (Opcode `0x82`).
   - Removed `CryptBinaryToStringA` dependencies, neutralizing the base64 encoding/decoding overhead.
   - Reduced server-side latency significantly.

2. **60 FPS Video Streaming & Surveillance**
   - Bumped up the core loop from 5Hz (`Sleep(200)`) to 60Hz (`Sleep(16)`).
   - Real-time client-side rendering implemented directly onto HTML5 `<canvas>` via `ArrayBuffer` handling.

3. **Walkie-Talkie Two-Way Audio Communication**
   - Capture live microphone input in the browser using `getUserMedia` and `ScriptProcessor`.
   - Transmits audio buffers seamlessly to the C server.
   - Win32 `waveOut` audio playback architecture seamlessly decodes and outputs audio chunk streams.

4. **Resource & Build Enhancements**
   - Updated executable metadata strictly to `4.6.0.0` within `src/calc_avg.rc`.
   - Optimized UI Walkie-Talkie toggles via CSS/JS logic.
